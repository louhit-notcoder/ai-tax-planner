# Security verification harness

Automated object-level authorization, privilege-escalation, locked-case mutation,
unknown AI tool, password-protected PDF and export-gating tests live in
`backend/tests_v3/test_production_api_v3.py`.

Before launch, run an independent authenticated penetration test covering:

- tenant and case IDOR/BOLA;
- role invitation and MFA bypass;
- malicious PDF/XLSX/image uploads;
- direct object-storage access;
- prompt injection in every supported document adapter;
- SSRF, request smuggling and proxy/header trust;
- session fixation, refresh-token replay and token revocation;
- race conditions around fact approval, computation approval and case locking;
- export snapshot tampering and schema/utility validator bypass;
- cloud IAM, S3 bucket policies, KMS access and backup restoration.

An external penetration-test report cannot be generated by this repository and is
a mandatory release gate.

# CA golden-case certification guide

Each case must be anonymised and contain:

- immutable input-fact JSON and evidence inventory;
- expected income-head and schedule values;
- old/new regime result where applicable;
- tax, surcharge, cess, interest, credits, refund/payable;
- expected ITR form and official utility result;
- trusted CA-software comparison;
- reviewing CA identity, review date and signed disposition;
- any accepted legal interpretation or known reference-tool difference.

Minimum target before limited production:

| Case family | Target |
|---|---:|
| Salary only | 100 |
| Multiple employers | 75 |
| Interest/dividend | 75 |
| House property | 100 |
| Regime comparisons | 100 |
| Listed equity | 150 |
| Selected mutual funds | 100 |
| Loss set-off | 75 |
| Tax payments | 75 |
| Boundary cases | 200 |
| Unsupported blockers | 100 |

Run exact comparisons—not semantic similarity. Every variance needs a classified
resolution: Green Papaya defect, source-data ambiguity, official utility version,
rounding, reference-software difference or unsupported scenario.
